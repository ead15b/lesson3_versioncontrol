Highlight #1 Version control system

Version control is when there is one main repository for the project files. Version control systems are in charge of automatically fixing any changes made, noting who changed them and knowing if multiple people were working on the file at the same time.The benefits of using this system would include files not being able to be overwritten, multiple people being able to work on one project in a file at the same time,and will constantly back up your work so that you will not have to worry about starting all over again. Version control systems use other programs to help them function including remote and local repositories.

Highlight #2 Local and Remote Repositories

A remote repository is used by multiple members in a team on a single server. Local repositories is only used by one person on a comupter. The only two ways to create a local repository is by creating a new one or copying a remote one onto your server. Therefore you can use a remote repository to go in between local repositories. You can put files that are in a directory into a repository and track the status of all files.

Highlight #3 GIT

Git is used to work on large and small projects it is a type of version control system. The difference is that it is a distributed control system. The benefits of using this control system would be that it runs smoothly and fast and will prevent files from being overwritten accidently.To use GIT you have to use the GIT commands such as: git init, git log, etc.
